# 微信公众号写作助手

**WeChat-Awesome — 你提供素材，它负责结构**

---

## 这是什么

一个把你的素材变成公众号爆文结构的编辑器。

你投喂文章、数据、笔记、截图，它用经过20+篇实战验证的规则重组内容，输出三样东西：

```
1. MD正文         → 粘贴到 ilang.cn/md 一键复制到微信编辑器
2. 封面图提示词    → I-Lang格式，丢给任何AI出图工具生成2.35:1封面
3. 自查报告        → 指纹词、em-dash、反问句、表格数量、品牌脱敏全部检查
```

## 这不是什么

不是"帮我写一篇爆文"的生成器。没有素材的请求会被直接拒绝。

不需要傻瓜用户。你得自己找素材、自己定角度、自己补个人经历。工具只负责结构和规范。

---

## 内置了什么规则

**写作铁律（每篇必须遵守）：**

• 开头3句话内必须有具体数字、事件或问题
• 数字压倒形容词（"天价门票" → "2999元门票"）
• 每1000字至少2张真实数据对比表格
• 每段不超过3行（移动端铁律）
• 用 **加粗文字** 分段，不用 ## 标题
• bullet用 • 不用 -
• 禁止 em-dash（— 和 –）
• 第一人称"我"，博主视角
• 结尾反问+回扣标题+评论区疑问句引导
• 长文在40%-60%处插钩子
• 说完就走不加戏

**品牌脱敏（微信专用）：**

| 原名 | 写法 |
|------|------|
| Claude | A社 |
| Claude Code | CC |
| OpenAI | O社 |
| Telegram | 电报 |
| VPN | 隐私工具 |

**平台合规（血泪教训）：**

• 文内不放URL（限流），外部链接放"阅读原文"
• 不放个人微信号（导流违规）
• 不引导点赞/转发/收藏（可能限流或封号）
• 被毙过的文章不原文重发（有指纹）

**自查清单（初稿完成后自动执行）：**

| 检查项 | 标准 |
|--------|------|
| AI指纹词 | 0个 |
| em-dash | 0个 |
| 反问句 | ≥2个 |
| 数字vs形容词 | 数字多 |
| 段落长度 | ≤3行 |
| 表格 | 每1000字≥2张 |
| 结尾 | 反问+回扣标题 |
| 品牌脱敏 | 全部完成 |
| 个人经历标注 | ≥1处[📝] |
| 文内URL | 0个 |

## 封面图提示词

每篇文章自动生成I-Lang格式的封面图提示词：

• 比例：2.35:1（公众号宽幅封面），中心构图裁切1:1不丢主体
• 风格：深色背景 + 明亮主体 + 扁平插画 + 科技配色
• 约束：无文字、无logo、无品牌名、无真人面孔
• 变量：根据文章核心观点自动生成主题和情绪

用户拿提示词去GPT/Midjourney/DALL-E/任何AI出图工具生成封面。

## 使用方法

1. 打开 `prompt.md`，复制全文到任何AI
2. 投喂你的素材（文章链接、数据、笔记、截图）
3. AI确认角度 → 你选
4. AI出提纲 → 你确认
5. AI写初稿 + 自动自查 → 你审改（补个人经历、补口语）
6. 输出MD → 粘贴到 ilang.cn/md 排版 → 一键复制到微信编辑器

如需去AI味处理，初稿完成后使用 [DeAI 技能](../DeAI/SKILL.md)。

## 不做的事

• ❌ 不接受无素材请求（"帮我写一篇XX"直接拒绝）
• ❌ 不编造数字、引语、个人经历（不知道就标注让用户补）
• ❌ 不引导点赞/转发/收藏
• ❌ AI不硬塞俚语（AI加的俚语本身就有AI味）
• ❌ 不做矩阵号/批量操作
• ❌ 不在文章内放URL
